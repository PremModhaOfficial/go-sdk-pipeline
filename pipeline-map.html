A multi-agent, NFR-driven, self-improving SDK code generation pipeline

Turns a human-authored TPRD (Technical Product Requirements Document) into a reviewed, tested, benchmarked Go SDK package on a dedicated branch of a target repo. Runs five phases orchestrated by phase-lead agents, with adversarial "devil" reviewers, mechanical guardrail checks, seven human-in-the-loop gates, and a Phase 4 feedback/self-learning loop that patches agent prompts and existing skills across runs — never creating new skills/agents itself (those require human PRs).

Phases — detailed view

HITL gates — human checkpoints

Seven primary gates + H0 preflight + H7b mid-impl surface co-owned symbols. Default timeouts: 24h / 48h / 72h with action-on-timeout varying by gate.

Loops — iteration points

Five distinct loops across local (within phase) and global (cross-run) scope.

Agents — all 33

Leads orchestrate phases; devils review (read-only); feedback agents analyze and patch. Click any card for invocation hierarchy, token cost, and lifecycle.

Skills — 52 total

31 SDK-native + 19 ported + 2 ported-with-deltas. Invoked by agents via the Skill tool; patched (minor bump) by learning-engine across runs.

MCP servers — 6

Every MCP is an enhancement, not a correctness dependency. G04 checks reachability at phase start; agents degrade to JSONL/Grep fallbacks on unavailability. Pipeline never halts on MCP failure.

Scripts & guardrails

40+ guardrail shell scripts enforced at phase checkpoints + a Python JSONL→neo4j ETL bridge.
