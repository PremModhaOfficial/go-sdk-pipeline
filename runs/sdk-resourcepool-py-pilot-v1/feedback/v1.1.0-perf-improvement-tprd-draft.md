## 1. Purpose

Raise `Pool.acquire` contention throughput from the v1.0.0 measured ceiling of **458k acq/sec** (32 acquirers × max_size=4) to **≥ 1M acq/sec** (~2.2× target) by replacing the `asyncio.Lock` + `asyncio.Condition` slot-accounting state machine in `_pool.py` with a lock-free counter + `asyncio.Event`-array wakeup pattern.

## 2. Goals

- `Pool.acquire` contention throughput ≥ **1,000,000 acq/sec** at 32:4 best-of-3 (~2.2× v1.0.0 458k ceiling).
- Public API surface unchanged — every TPRD v1.0.0 §5 symbol keeps its current signature (`PoolConfig`, `Pool`, `AcquiredResource`, `PoolStats`, `PoolError` family).
- `mypy --strict` stays clean; `ruff` stays clean.
- Coverage stays ≥ 90% on the package.
- All 81 v1.0.0 tests stay green (zero regressions on cancellation, timeout, aclose, hook-panic, leak, integration).
- Acceptance bench: `test_contention_throughput_meets_1M_per_sec_budget` passes best-of-3.

## 3. Non-goals

- No public-API breaking change. (semver: minor bump 1.0.0 → 1.1.0; no major.)
- No new runtime dependencies (TPRD v1.0.0 §4 zero-deps invariant preserved).
- No thread-pool variant.
- No additional acquire-method shapes.
- No change to the cancellation contract per concurrency-model.md §3 (must continue to roll back slot reservation under all `BaseException` paths).
- No change to the leak-harness contract (must remain clean per `tests/leak/test_no_leaked_tasks.py`).

## 6. Internal redesign sketch

Replace the v1.0.0 `_pool.py` slot-accounting machinery:

| v1.0.0 (current) | v1.1.0 (proposed) |
|---|---|
| `self._lock: asyncio.Lock` | dropped from fast path; only used at `aclose` for the drain barrier |
| `self._slot_available: asyncio.Condition(self._lock)` | replaced by `self._slot_events: list[asyncio.Event]` (one per slot) + a free-event index |
| `self._idle: deque[T]` | kept (`pop()` and `append()` remain O(1)) |
| `self._created/in_use/waiting: int` (mutated under lock) | mutated directly under GIL (single-event-loop invariant means no thread-race) |
| Acquire critical section: ~2 µs (lock + condition wait_for + counter mutations) | Target ~0.5 µs (deque.pop + GIL-protected counter + Event.wait if blocked) |

**Wake-up strategy**: a parked acquirer `await`s the next free `asyncio.Event`; releaser sets the next-pending event. No condition-variable predicate-rescan; direct point-to-point wake.

**Cancellation correctness preserved**: the per-event `await` is cancel-safe by stdlib contract (Event.wait raises CancelledError). The `try/finally` rollback pattern from `concurrency-model.md §3` still applies — but the `_created -= 1` rollback now happens directly under GIL (no lock to take).

**Risks** (see §10).

## 10. Risks

| Risk | Mitigation |
|---|---|
| Cancellation rollback under the lock-free path leaks `_created` counter | Re-run all 4 cancellation tests at every iteration; add a fuzz test that randomly cancels mid-acquire 1000× and asserts `created == in_use + idle` invariant. |
| `asyncio.Event` per-slot allocation reverses the alloc-budget gain | Pool the events: `_slot_events: list[asyncio.Event]` is allocated once at `__init__`, reused indefinitely. |
| Releaser-to-acquirer wakeup race (event already set when next acquirer arrives) | Standard pattern: acquirer takes the slot index FIRST (under GIL), then awaits its event; releaser sets the event for the slot it released. |
| `aclose` drain semantics differ from v1.0.0 | The drain barrier still uses `self._lock` (cold path); the fast path is lock-free but `aclose` is allowed to be slow. |
| GIL-protected counter mutations break under free-threading (PEP 703 / Python 3.13t no-GIL) | Out of scope for v1.1.0; document in §3 Non-goals. v1.2.0 considers atomic int via `multiprocessing.Value` if needed. |
| Public API still must match v1.0.0 — refactor is internal only | semver-devil verifies; `[stable-since: v1.0.0]` markers preserved on all 9 public symbols. |
